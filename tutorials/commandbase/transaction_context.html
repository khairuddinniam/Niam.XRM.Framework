TransactionContext

From OperationBase you can access the Context object. This object have access for this:

  1. Initial
  2. Input is same with InputParameters["Target"] that holds only changes in this target entity.
  3. Reference is merge between Initial data and Input.
  4. PluginExecutionContext is an implementation of IPluginExecutionContext in this Plugin Step.
  5. Plugin holds SecureConfig and UnsecureConfig for the PluginStep.
  6. Service is an implementation of IOrganizationService in this target entity.
  7. SystemService is an implementation of IOrganizationService on behalf of Admin.
  8. TracingService is an implementation of ITracingService.
  9. ServiceFactory is an implementation of IOrganizationServiceFactory.
  10. ServiceProvider is an implementation of IServiceProvider.
